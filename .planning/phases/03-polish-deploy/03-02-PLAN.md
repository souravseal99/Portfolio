  <action>

<task type="auto">
  <name>Deploy to Vercel</name>
  <files>[]</files>
    <action>
    Deploy to Vercel using one of these methods:

    **Option A - GitHub Integration (recommended):**
    1. Push code to GitHub if not already pushed
    2. Go to vercel.com and sign up/login
    3. Click "Add New" -> "Project"
    4. Import your GitHub repository
    5. Configure: Framework Preset = Next.js (auto-detected)
    6. Click "Deploy"
    
    **Option B - Vercel CLI:**
    1. Run `npx vercel` in project root
    2. Login to Vercel if prompted
    3. Set up and deploy (answer prompts)
    4. Note the deployed URL
    
    After deployment, you will have a live URL like: https://souravseal.vercel.app
  </action>
  <verify>
    Visit your Vercel deployment URL and verify:
    - Page loads without errors
    - SEO metadata visible in page source
    - All sections render correctly
  </verify>
  <done>
    Website is live at a public Vercel URL (e.g., https://souravseal.vercel.app)
  </done>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Performance verification via Lighthouse</what-built>
  <how-to-verify>
    1. Open your deployed Vercel URL in Chrome
    2. Open Chrome DevTools (F12)
    3. Go to Lighthouse tab
    4. Select "Mobile" or "Desktop" mode
    5. Click "Analyze page load"
    6. Verify Performance score is green (90+)
    7. Verify "Largest Contentful Paint" < 2.5s
    8. Verify "Total Blocking Time" < 200ms
    
    **Expected result:** Page loads in under 3 seconds with 90+ performance score
  </how-to-verify>
  <resume-signal>Type "approved" if Lighthouse shows < 3s load time, or describe issues if performance needs improvement</resume-signal>
</task>
